# Cyclo Framework

## Overview

**Cyclo Framework** is a working name for a **cyclic development** practice: short build–measure–document loops that keep hardware-in-the-loop stacks honest. It is not a product; it is a checklist and folder contract I reuse across projects.

## Motivation

Long integration cycles hide coupling. Tightening the loop surfaces scheduling issues, frame mismatches, and brittle assumptions while they are still cheap to fix.

## Technical Scope

- Weekly integration goals sized to fit **two afternoons** of focused bench time  
- Standard experiment cards: hypothesis, setup, metrics, outcome, next tweak  
- Promotion path from **Tech Log** bullets to **Article** outlines at month end

## What I Learned

- The bottleneck is rarely compiler speed; it is **unclear success criteria** before touching the robot.  
- A lightweight schema beats a heavy “process” document when everyone actually fills it in.
